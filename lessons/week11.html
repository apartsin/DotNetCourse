Common misconception to confront.

Students often think: high coverage proves quality.
Set it straight: coverage says code ran. Quality comes from assertions tied to behavior and risk.

Check for understanding
What should an integration test prove?
That multiple real components cooperate correctly across a boundary such as API plus database.
What is an architecture test useful for?
Preventing accidental dependency violations as the codebase grows.
Key takeaways.
  • Tests are design feedback.
  • Integration tests protect boundaries.
  • Architecture tests keep structure honest.
